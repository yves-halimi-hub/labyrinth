# Unity Editor Integration

[Up to Scripts](../README.md) | [Game repository](../../../README.md)

Editor-only bridge from LabyMake output to runtime assets.

## Files

- [`EFYVPixelArtImporter.cs`](EFYVPixelArtImporter.cs): validates metadata (via the shared backend `TryValidateAtlasMetadata`; item #6 attachment records via the shared `TryValidateAttachments`, rejecting with the offending index; item #5 tileset manifests via the shared `TryValidateTilesetManifest`, validated against the sibling atlas block), configures pixel-art slicing, maps schema properties from the shared `Shared.AssetSchemaFieldManifest` table, creates typed assets, links directional imports, and stores converted attachment records on the schema-backed base (absent flips resolve to false; a document without attachments stores null). Item #5: a document carrying a tileset manifest materializes as `TilesetAssetData` (tile size, manifest names, and the sliced sprites in tile-id order) regardless of which `GameAssetData`-family factory its assetType names. Every rejection logs a per-cause error; unknown schema keys are logged, never silently dropped - and (item #33b) additionally parked on the asset's string-keyed custom-property store as raw JSON text so custom-registered consumers can read runtime-registered designer fields; an import without unknown keys clears stale entries. Item #27: `OnPostprocessAllAssets` dedupes each metadata file with its sibling PNG, imports each once (`ImportEFYVAsset` only `SetDirty`s its asset and reports whether it changed anything), then coalesces `AssetDatabase.SaveAssets()` into ONE flush for the whole group — and skips the flush entirely when nothing was created/dirtied. The synchronous forced texture re-import (`EnsureTextureImportIsCurrent`) is kept ONLY where it is required — the caller reads the freshly sliced sprites immediately after — and only fires when the texture settings/slices are actually stale.
- [`EFYVMapImporter.cs`](EFYVMapImporter.cs) (item #5): the `.efyvmap` counterpart — parses published binary maps through the backend `FastMapImporter.TryParse` tri-state (per-cause missing/malformed/unsafe-stem/type-mismatch rejections via `Game.MapImporter` messages), creates or updates the `MapAssetData` at `<stem>_Map.asset`, and links the sibling tileset asset (`<tilesetName>_Data.asset`, warning when it has no sliced sprites yet — publish/import the tileset first, then the map).
- [`EFYVRawArtWatcher.cs`](EFYVRawArtWatcher.cs): `[InitializeOnLoad]` debounced `EditorApplication.update` poller over `Assets/RawArt` that imports freshly published art without waiting for Unity to regain focus (works in Play Mode). The debounce/coalescing state machine (`RawArtChangeTracker`) is plain C# and unit-tested headlessly.
- [`EFYVLiveDebugBridge.cs`](EFYVLiveDebugBridge.cs): coalesces Play Mode refreshes and updates matching scene entities/props. Item #4: it also records the most recently imported/refreshed asset (`LastRefreshedAsset` + a `RefreshVersion` counter, tracked in edit mode as well) so the spawn palette can auto-offer the newest import.
- [`EFYVSpawnPaletteWindow.cs`](EFYVSpawnPaletteWindow.cs) (item #4): the Play-Mode debug spawn window. Lists the imported schema-backed assets under `Assets/RawArt`, auto-offers the most recently imported one (the bridge seam above), and one-clicks the selection into the running game through the runtime `DataToPrefabFactory` (which picks the archetype template prefab under `Assets/Prefabs/DebugTemplates`, spawns it through `PoolManager`, and binds the asset so the flipbook + hurtbox go live). Placement is player-relative (spawn at the player position plus a configurable offset, or world origin when no player exists) - the simpler of the two options, no scene-view raycast. All list/selection logic lives in the plain, headlessly-tested [`SpawnPaletteModel`](../Core/Spawning/README.md); the window is a thin IMGUI shell plus the editor-only asset discovery and prefab loading.
- [`EFYVHitboxGizmo.cs`](EFYVHitboxGizmo.cs): an ALWAYS-ON hitbox overlay (item #14). The pixel-to-local-units conversion now lives in the runtime `EntityHitboxGeometry.TryGetLocalBounds` (shared with the gameplay `BoxCollider2D` sync — one implementation, not two); this gizmo draws the frame-specific wire cubes. In edit mode it previews the designer's `HitboxPreviewFrame`/`HitboxPreviewFacing`; in Play Mode it follows the LIVE facing and the current animation frame of the item #13 runtime flipbook.

The metadata filename stem and `entityName` or `assetName` must pass the backend safe-stem policy; a file without either identity is rejected (no `UnknownEntity` fallback). Importer factories are generated from `EFYVLabyrinthConfig.LabyMake.Schema.BuiltInAssetRegistrations`: a registration whose concrete class exists imports as that class, otherwise it falls back to the factory of its `baseAssetType` (which exporters now embed in the `.efyvlaby`). Documents carry a top-level `documentVersion`; version-absent legacy files read as version 1, and the importer accepts the whole supported range `[MinSupportedDocumentVersion .. CurrentDocumentVersion]` (currently 1..5 — version 2 added the optional atlas animation timing/playback fields, which the importer resolves into `EntityAnimationMetadata` with fps/full-range defaults; version 3 the optional per-animation effect descriptors; version 4 the optional top-level attachment records; version 5 the optional tileset manifest block).
