# Entities

[Up to runtime core](../README.md)

The gameplay object hierarchy and its packed global iteration lists.

## Hierarchy

- [`GameEntity.cs`](GameEntity.cs): Unity component caching, pool identity, spawn state, and the scene-placed registry. Entities dropped directly into a scene (never pool-spawned) register as pending in `Awake` (pool factory clones and prefab assets are excluded); `SpawnManager.Update` and the map switch promote them via `OnSpawn` into the per-type packed lists so they tick, are targetable, and are cleaned on map switch. `PlayerController` opts out via `TracksAsScenePlaced`.
- [`Faction.cs`](Faction.cs): the combat sides. `Player` is the zero value so unowned weapons and projectiles default to the historic player-side behavior.
- [`LivingEntity.cs`](LivingEntity.cs): health, authored stats, directional sprites, and damage. `TakeDamage` clamps negative amounts to zero — damage never heals; only `Heal` restores health. Item #13: it plays the CURRENT facing's imported animation as a runtime flipbook over `EntityFacingImportData.Frames`, advanced by the central update loops (`Enemy.Tick` / `PlayerController.Update` via `TickFlipbook`) with no per-frame allocations — honoring the imported per-frame durations (`0` = inherit the animation fps), the clamped loop range, and ping-pong. `ApplyDirectionalSprite` resolves this flipbook (replacing the old static frame-0 swap), preserving playback progress across a facing change when the same clip keeps playing and restarting it on a state change; a facing with no imported frames falls back to the hand-authored static directional sprite. `PlayAnimation` is the animation-state seam (idle/walk/attack) — a state takes effect only when an authored animation carries the matching `Name`, else the first clip plays; movement drives walk/idle (`Enemy.Tick` when chasing, `PlayerController` on input) and attack is an available seam with no auto-revert. Item #14: the CURRENT facing + flipbook frame's `Hurtbox` record (`Game.Hitbox.HurtboxType`) drives a cached hand-placed `BoxCollider2D` (offset/size in local units via `EntityHitboxGeometry.TryGetLocalBounds`), re-synced when the facing or frame changes — frame-specific, and frames without a `Hurtbox` inherit the last synced bounds; an entity without a `BoxCollider2D` (or facing without imported hitboxes) is a safe no-op. Item #7: it minimally interprets the asset's authored effect descriptors against the `SpriteRenderer` color — `OnSpawn` resets the color then fires `OnSpawn`-tagged effects, a positive post-clamp hit fires `OnDamaged`-tagged ones; `tint` persistently recolors (and becomes the flash-restore target), `flash` temporarily recolors for its duration with the countdown ticked by the central loops (`Enemy.Tick`, `PlayerController.Update` via `TickAuthoredEffects`); matching scans the CURRENT facing's imported animations (narrowing effect matching to the playing flipbook clip is a possible future refinement). `particleHook` descriptors are stored on the asset but NOT interpreted yet (deferred until a particle pipeline exists). Item #6: it is also the minimal attachment consumer — applying source data stores the asset's imported sub-element attachment records (`AuthoredAttachments`, with `CountAttachmentsForFrame`); the attachment pixels are already flattened into the imported atlas, and rendering dynamic sub-element sprites from these records is deferred until a sprite pipeline for bank sub-elements exists.
- [`Enemy.cs`](Enemy.cs) and [`BossEnemy.cs`](BossEnemy.cs): scaling, targeting, packed enemy membership, and phases. Enemies stop chasing and stop contact-attacking a dead player (custom non-player targets keep working).
- [`PlayerController.cs`](PlayerController.cs): input, invulnerability, experience, session currency, timed buffs, and projectile loop. `Initialize` folds the persisted meta-progression multipliers (`SaveManager.GetCombinedStatsForToon`: `MaxHealth`, `MoveSpeed`) into the base stats — non-finite or non-positive slots fall back to the neutral 1x; `ReinitializeForToon` re-runs the fold for a selected toon. `ApplyTimedBuff` registers centrally ticked buffs that revert on expiry (re-application refreshes, never stacks). `Die` is idempotent: it latches `IsDead`, despawns, then raises the static `OnPlayerDied` event exactly once so managers can react to game over; `OnSpawn` clears the dead state and the buff list. Session coin addition saturates at `int.MaxValue`.
- [`Projectile.cs`](Projectile.cs): normalized movement, lifetime, piercing, and packed projectile membership. Damage follows the firing weapon's `OwnerFaction` (player-owned hits enemies, enemy-owned hits the player); `OnSpawn` resets the pierce counter and the per-collider component-lookup memo for pool reuse.
