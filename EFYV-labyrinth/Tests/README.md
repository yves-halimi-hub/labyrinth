# Game And Editor Verification

[Up to game repository](../README.md)

External tests compile all backend and game/editor C# sources without placing test code under Unity's `Assets` directory.

## Files

- [`EFYVGame.Verification.csproj`](EFYVGame.Verification.csproj): dependency-free .NET 10 (`net10.0`) console verification project.
- [`UnityStubs.cs`](UnityStubs.cs): controlled UnityEngine and UnityEditor API surface for headless tests, including the monotonic `GameObject.GetComponentCalls` and `Object.FindObjectOfTypeCalls` instrumentation counters (assert deltas only; they are never reset). `Instantiate(component)` clones the whole GameObject (all sibling components), mirroring Unity so pooled clones keep their renderer/collider. It also carries a minimal `EditorWindow`/IMGUI signature surface so the editor-only `EFYVSpawnPaletteWindow` compiles (the window is never driven by a test; its logic lives in the plain `SpawnPaletteModel`).
- [`GameDataEditorTests.cs`](GameDataEditorTests.cs): schema assets, importer, hitboxes, live refresh, the shared schema-field manifest end to end, PropEntity walkable/trap-damage schema reads, the baseAssetType/documentVersion import contract (supported-range acceptance, future/below-floor rejection), the item #10 atlas timing/playback metadata import (per-frame durations with the inherit sentinel, resolved loop range and ping-pong defaults, broken-timing rejection), and the RawArt watcher debounce/poll state machine.
- [`GameRuntimeTests.cs`](GameRuntimeTests.cs): entities, projectiles, weapons, pools, vectors, viewports, and the singleton negative-cache/invalidation contract.
- [`GameManagerTests.cs`](GameManagerTests.cs): saves, progression, maps, props, achievements, manager states, the runtime upgrade-loop wiring (weapon-pool grants, chest/level-up application, special-attack phase gating), and the meta-progression fold + timed buffs + kill/survival achievement triggers.
- [`EntitiesDeepTests.cs`](EntitiesDeepTests.cs): player i-frames and progression, game-over state and the `OnPlayerDied` broadcast, enemy scaling and swap-list fuzz, boss and eye-bearer spawns, projectile reuse (pierce reset, trigger memo, faction gate), props (animation-speed sanitization), merchants (interaction-request event + explicit purchase API), and sarcophage branches.
- [`WeaponsDeepTests.cs`](WeaponsDeepTests.cs): cooldown state machine, wand targeting, aura/melee/orbital boundaries (tick-scaled damage and knockback), seeded drop and splash placement, pool contracts (nearest targeting, typed-prefab path, type-check-before-activation), faction ownership and dead-player targeting, evolutions, and powerups.
- [`ManagersDeepTests.cs`](ManagersDeepTests.cs): PRNG and loot reference models (chests, coins, and XP gems), achievement bitmasks, save binary layout and debounce, stat rules, spawn accumulator and placement, pool scheduling (idempotent GameObject despawn, key-validated before deactivation), viewport ring buffer, map switching, upgrade penalties, game-over manager reactions, scene-placed entity lifecycle, and pool prewarm wiring.
- [`DataEditorDeepTests.cs`](DataEditorDeepTests.cs): schema bit preservation, FNV hash reference models, achievement sync, facing import retention, hitbox gizmos, live debug bridge, texture reimport rules, and adversarial import paths.
- [`EffectsRuntimeTests.cs`](EffectsRuntimeTests.cs): the item #7 runtime surface — importer conversion of per-animation `effects` arrays (shared defaults for absent optionals, copy isolation, end-to-end documentVersion-3 import, unknown-type rejection with the `AnimationEffects` cause) and the `LivingEntity` flash/tint interpretation (spawn tint persistence, hurt flash over tint with the central-loop countdown and tint-restore, zero-damage immunity, pooled-reuse color reset, particleHook stored-only, renderer-less/facing-less no-ops, `Enemy.Tick` integration).
- [`AttachmentRuntimeTests.cs`](AttachmentRuntimeTests.cs): the item #6 runtime surface — importer conversion of the top-level `attachments` array (absent flips resolving to false, null/empty producing no records, end-to-end documentVersion-4 import onto the schema-backed base, stale-record clearing on legacy reimports, invalid-record rejection with the offending index logged) and the minimal `LivingEntity` consumer (records stored on data application with per-frame counting; rendering dynamic sub-element sprites stays deferred — the atlas already contains the flattened pixels).
- [`CustomFieldsRuntimeTests.cs`](CustomFieldsRuntimeTests.cs): the item #33b runtime surface — `SchemaBackedAssetData`'s string-keyed custom-property store (ordinal lookup, invariant-culture typed parse-on-read accessors, mismatched-array rejection, null-clears semantics) and the importer path end to end (unknown `.efyvlaby` properties keys parked as raw JSON text with strings unquoted, identity/manifest keys excluded, the batch-2 unknown-keys warning still logged, stale entries cleared on a reimport without unknown keys).
- [`GameBootstrapTests.cs`](GameBootstrapTests.cs): scene-composition bootstrap for `Assets/Scenes/Labyrinth.unity` — deterministic placeholder sprite generation, tile-palette wiring, authored-data-wins guards, and the inactive-enemy-template-to-pooled-clone stat path (schema block, chase, damage).
- [`SpawnPaletteRuntimeTests.cs`](SpawnPaletteRuntimeTests.cs): the item #4 debug spawn palette + data-to-prefab factory — archetype resolution (base archetypes, custom registered types falling back to their base, plain-`EntityData`/null rejection, boss-before-enemy ordering, display-name resolution), the pooled spawn + `LoadData` bind that lights up the flipbook + hurtbox on a living-entity clone and registers an animated prop in the central loop (bind runs before `OnSpawn`), unknown/missing-template/null rejections adding nothing, the `SpawnPaletteModel` list/refresh/selection state machine (dedup, selection preservation, one-shot auto-offer of the newest import), and the `EFYVLiveDebugBridge` last-refreshed seam.
- [`MapRuntimeTests.cs`](MapRuntimeTests.cs): the item #5 game surface - EFYVMapImporter end to end (.efyvmap through the real FastMapExporter envelope into MapAssetData with tiles/props/id copied, tileset-asset linking with the missing-sprites warning, update-in-place, malformed/missing/unsafe-stem/type-mismatch rejections, batch postprocessor routing), tileset import end to end (documentVersion-5 manifest into TilesetAssetData with sliced sprites in tile-id order, shared-gate rejection of atlas-mismatched manifests, existing-asset type mismatch), and MapViewportController.LoadMapData (serialized-slot-then-loaded-asset resolution skipping unloadable entries, grid rebuild on dimension change, tilePalette fed from the imported tileset, explicit procedural fallback for unknown ids, and the Start()-time default-map load that DoorProp/MapManager switching drives).
- [`test_mock_exporter.py`](test_mock_exporter.py): PNG, metadata, repeatability, and path-attack tests.
- [`test_mock_exporter_deep.py`](test_mock_exporter_deep.py): byte-exact PNG references, chunk/CRC/zlib framing, metadata text and type contracts, publication ordering, GUID temp-name and exclusive-create semantics, and Windows-edge name policies.
- [`test_config_contract.py`](test_config_contract.py): cross-language contract guard extracting field names, extensions, facing/type strings, temp-name rules, the atlas format version, and the top-level documentVersion/baseAssetType contract from `EFYV-labybackend/Core/Data/EFYV-LabyrinthConfig.cs` and asserting `MockExporter.py` output matches.
- [`test_unity_project_assets.py`](test_unity_project_assets.py): static validation of the openable Unity project — pinned editor version and ProjectSettings flags, Packages/manifest to local-package cross-reference, asmdef graph and unsafe flags, `.meta` coverage plus GUID uniqueness and the `md5("efyv:" + path)` scheme, scene fileID/GUID referential integrity and wiring, the item #4 debug-template prefab integrity (local fileIDs resolve, script GUIDs resolve to `.cs` metas, external GUIDs builtin/authored, and each template wires exactly the intended component scripts), EditorBuildSettings agreement, BCL-compat package contents, and legacy input axis names against the shared config.

```powershell
dotnet run --project EFYVGame.Verification.csproj -c Debug
dotnet run --project EFYVGame.Verification.csproj -c Release
python -m unittest discover -s . -p "test_*.py" -v
```

The stubs deliberately model only APIs used by this source tree. Passing here does not replace tests inside the real Unity engine for serialization, rendering, physics, or AssetDatabase behavior.
