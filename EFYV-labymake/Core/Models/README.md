# Models

[Core](../README.md) | [Repository](../../README.md) | [Logic](../Logic/README.md) | [Persistence](../Persistence/README.md)

The model layer separates mutable designer state from immutable preview/export state. Packed scalar data uses backend schema blocks; owned collections and pixel arrays remain explicit C# objects.

## Mutable authoring graph

- [`EFYVProject`](EFYVProject.cs) holds the target asset type, typed property dictionary, Unity project path, canvas size, deterministic designer seed, and ordered animations. The `CanvasWidth`/`CanvasHeight` setters are internal: assigning a new size never resizes already-allocated layer buffers, so hosts resize through `DesignerSession.ResizeCanvas` (one undoable command that rebuilds every frame); internal callers only assign them while the frame graph is rebuilt or empty. Item #33: the optional `Directional` state (internal setter, null = plain project) marks a linked 4-direction project; `GetFacingAnimations(facing)` routes the active facing to `Animations` and the rest to the parked sets, and the internal `EnumerateAllAnimations` spans every facing for whole-project operations.
- [`DirectionalState`](DirectionalState.cs) (item #33) is the linked 4-direction authoring state: the active facing name plus one parked animation list per INACTIVE facing (the active facing's animations stay in `EFYVProject.Animations`, so every existing tool/command/undo path keeps working). `Switch` moves animation sets BY REFERENCE between the live list and parking — a facing switch never clones frames. Hosts drive it through `DesignerSession.EnableDirectionalAuthoring`/`SwitchFacing`/`GenerateMirroredFacing` (all undoable) or create one via `ToolbarAPI.CreateNewLinkedDirectionalProject`.
- [`AnimationState`](AnimationState.cs) owns a name, positive FPS, ordered frames, the item #10 playback tags: `LoopStartFrame` (>= 0), `LoopEndFrame` (`FullRangeLoopEnd` -1 = "the last frame"), and `PingPong`, plus the item #7 `Effects` list of authored runtime-effect descriptors. The raw tag values are authoring intent and may go stale after frame removals — consumers (preview, export) clamp via `AnimationSnapshot.EffectiveLoopStart/End`. `Clone` copies the tags and shares the immutable effect descriptors (isolated list).
- [`EffectDescriptor`](EffectDescriptor.cs) (item #7) is one authored runtime effect: `EffectType` (`flash`/`tint`/`particleHook` wire strings), `Name` (required non-empty for particleHook), `Trigger` (non-empty seam tag, e.g. `OnSpawn`/`OnDamaged`), `ColorRgba`, `DurationMs`, and `Strength`. IMMUTABLE by design — the constructor is the single validation gate (`LabyMake.Effect` config bounds), so instances are always well-formed and clones/snapshots/undo commands share references. Hosts mutate through the undoable `DesignerSession` effect CRUD.
- [`Frame`](Frame.cs) owns fixed dimensions, ordered layers, keyed backend `HitboxData`, an item #10 `DurationMs` override (0 = inherit the animation FPS; positive values are bounded by `Animation.MinFrameDurationMs..MaxFrameDurationMs`), and the item #6 `Attachments` list of per-frame sub-element attachment records. `Clone` deep-copies layers, hitbox membership, the duration, and the attachments.
- [`Layer`](Layer.cs) stores contiguous RGBA32 pixels plus visibility and finite opacity clamped to `[0,1]`. Out-of-bounds reads are transparent and writes are ignored. The constructor sizes the buffer with a checked `width * height` multiply (matching `SubElement`), so dimensions whose product overflows throw instead of silently wrapping the buffer.
- [`SubElement`](SubElement.cs) is an owned, cloned pixel region used by the stamp tool and asset bank. Item #6 added a validated `PivotX`/`PivotY` anchor pixel (defaults to the center — the exact legacy stamp centering) and a default transform for new attachments: `DefaultOffsetX/Y`, `DefaultZOrder` (bounded by `Attachment.MinZOrder..MaxZOrder`), and `DefaultFlipX/Y`; all persisted in the `.efyvsub` version-2 header.
- [`SubElementAttachment`](SubElementAttachment.cs) (item #6) is one per-frame attachment: an immutable safe-file-stem `SubElementName` referencing a bank sub-element plus mutable `X`/`Y` (canvas position of that element's pivot), validated `ZOrder`, and `FlipX`/`FlipY`. The constructor is the validation gate; undo safety comes from `FrameEditCapture`/`FrameEditCommand` deep-copying whole lists.
- [`Selection.cs`](Selection.cs): `SelectionRegion` is the immutable pixel mask the selection tools produce (bounds pre-intersected with the frame; rectangle and even-odd lasso factories return `null` when nothing is selected), and `FloatingSelection` is the lifted/pasted masked buffer hovering over the canvas — it lives only inside a `DesignerSession`, which moves, composites, anchors, or cancels it.
- [`CanvasAnchor`](CanvasAnchor.cs) names the nine content-anchor positions for `DesignerSession.ResizeCanvas` (center splits odd differences toward the top-left).
- [`Tileset.cs`](Tileset.cs) (item #5): `TilesetSection` is the optional designer tileset — a list of `TilesetTile`s all at one fixed `TileSize` (`LabyMake.Tileset` caps). Each `TilesetTile` is a named mini-frame of raw straight-RGBA pixels; the constructor is the validation gate (non-blank bounded name, exact `TileSize²` pixel buffer, copied on construction). A tile's LIST INDEX is its FastGridMap short tile id — the exported tile-ID manifest and painted map cells both address tiles by that index, so removing/reordering tiles re-addresses maps (documented hazard). Hosts mutate through the undoable `DesignerSession` tileset CRUD; `EFYVProject.Tileset` is null when the project has none.
- [`MapSection.cs`](MapSection.cs) (item #5): the optional designer map — an immutable safe-stem `MapId` (the `.efyvmap` export stem), a validated `TilesetName` reference (empty = none), and an owned backend `FastGridMap` (`LabyMake.MapDocument` caps) whose cells start at `BlankTileId` (-1, below the runtime palette floor, so untouched cells render empty instead of aliasing palette tile 0). Hosts mutate tiles/props through the undoable `DesignerSession` map surface; `EFYVProject.Map` is null when the project has none.
- [`Palette.cs`](Palette.cs): `Palette` is a named, index-addressed, ordered swatch list (names are display labels, not unique keys; duplicate swatch colors allowed) with the palette-constraint nearest-color metric — squared Euclidean over the four straight RGBA channels, ties to the lowest index. `RecentColorRing` is the bounded most-recent-first ring of recently used colors with de-duplicating `Push`. `EFYVProject` owns one palette list and one ring; both persist in `.efyvmake`.

`PixelColor` packs channels as `R | G<<8 | B<<16 | A<<24`, matching backend export. A newly allocated canvas is transparent; authoring tools select their own opaque default brush color.

## Compositing invariants

`Frame.FlattenLayers` clears the destination, walks layers in list order, skips hidden/zero-opacity layers, verifies matching dimensions, and alpha-blends through backend `FastMemory`. The caller-provided overload requires exactly `Width * Height` pixels and enables buffer reuse.

## Immutable snapshots

[`ProjectSnapshot`](ProjectSnapshot.cs) captures project metadata, read-only property values, animation frame ranges, flattened frame pixels, and hitboxes. It also precomputes total frame and hitbox counts. `JsonElement` values are cloned; other unexpected reference values are reduced to strings. Item #10: `FrameSnapshot.DurationMs` carries each frame's raw duration override and `AnimationSnapshot` exposes the raw playback tags plus the clamped `EffectiveLoopStart`/`EffectiveLoopEnd` accessors every consumer plays/export against. Item #7: `AnimationSnapshot.Effects` is a read-only copy of the authored effect-descriptor list (immutable entries shared with the live model). Item #6: `FrameSnapshot.Attachments` is a read-only list of immutable `AttachmentSnapshot` copies (null model entries skipped) that export flattens into the atlas and emits as metadata.

Item #33: `ProjectSnapshot.CaptureFacing(project, facing)` captures ONE facing of a linked directional project — that facing's animation set plus the shared properties with the `facing` key overridden — so the export engine and live debug can publish all four suffixed pairs from one project; `Capture` keeps capturing exactly the live list.

Preview and export consume snapshots so later designer mutations cannot change in-flight work. Persistence uses a separate editable-project snapshot described in [Persistence](../Persistence/README.md).
