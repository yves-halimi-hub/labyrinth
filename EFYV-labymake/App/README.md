# EFYV LabyMake App

[Component overview](../README.md) | [Headless core](../Core/README.md)

The desktop editor shell for LabyMake: an Avalonia 11 / .NET 8 application that hosts the existing headless designer core. It contains **no authoring logic of its own** — every edit, validation, save, and undo flows through the same [`DesignerSession`](../Core/Logic/DesignerSession.cs) seams the verification suite exercises. The shell now carries the full editor panel set (item #3): an inspector, layers, animations/frames, problems, preview, live-debug, project-browser, and palette panel, docked as collapsible sections at the left edge.

## Run

From the repository root (first run restores the three Avalonia packages from NuGet):

```powershell
dotnet run --project EFYV-labymake\App\EFYVLabyMake.App.csproj -c Release
```

The project source-links `../Core/**/*.cs` and `../../EFYV-labybackend/Core/**/*.cs` exactly like the verification csproj, so there is no packaging step and no project-to-project reference. Only three NuGet packages are used: `Avalonia`, `Avalonia.Desktop`, `Avalonia.Themes.Fluent`. The console window stays open behind the editor and carries log output (MVP behavior).

## Layout

- `Program.cs` / `EditorApp.cs`: entry point and Avalonia application (Fluent dark theme, code-only UI — no XAML).
- `AppDefaults.cs`: shell-local presentation constants (tool names, swatches, the workspace color). Wire-format limits still come from the shared `EFYVLabyrinthConfig`; the checkerboard backdrop constants moved into `EFYVLabyrinthConfig.LabyMake.Overlay` with item #31.
- `State/`: UI-framework-free view-model layer (plain `INotifyPropertyChanged`, no MVVM framework).
  - `EditorShell.cs`: the single view-model. Owns the `DesignerSession`, the tool instances, the bindable status properties, the new-project/save/undo/redo/cancel operations, and (item #31) the overlay toggle set + the reusable `ViewportOverlaySettings` it stamps per render via `PrepareOverlayContext()` (tile-grid cell size from the map cell size in map mode or the tileset's `TileSize` otherwise; `AttachmentSources` from the loaded bank; map mode masks the frame-object passes because the composed map surface is a synthetic frame). The checkerboard starts on — the pre-#31 shell always drew it.
  - `ScreenPixelConverter.cs`: converts the viewport's straight-RGBA screen buffer (red in the low byte, the `PixelColor` layout) to opaque BGRA for the bitmap. Item #31 moved the checkerboard into the Core overlay passes, so this is now a plain swizzle that composites any remaining non-opaque pixel (outside the canvas, or inside it with the checkerboard toggled off) over the flat workspace color.
  - `ObservableObject.cs`: minimal `INotifyPropertyChanged` base.
  - Item #3 also extended `EditorShell` with the panel surface: the active-layer index (threaded into every drawing/selection tool and copy/paste — closing the batch-3 "always layer 0" deferral), layer/animation/frame CRUD wrappers, the problems list (from the item #27 lazy snapshot), preview load + playback accessors, the live-debug controls (Unity path, watch toggle, `Push to game`), palette + swatch CRUD + constraint mode, and the project-browser open/autosave-recovery flow — all forwarding to `DesignerSession` (no authoring logic added).
  - `ProblemFormatter.cs` / `PropertyFieldEditor.cs` / `LiveDebugFormatter.cs` / `PreviewStatusFormatter.cs`: pure, UI-framework-free formatting/parsing helpers the panels build on (problem messages + severity + focus location; inspector value ↔ text and edit-status text; live-debug and preview status lines). These plus `ScreenPixelConverter` are linked individually into the verification csproj and covered by `EditorAppStateTests.cs` (the batch-2 "App State layer tests" deferral). `EditorShell` and the panels stay App-only (they depend on `AppDefaults`/Avalonia).
  - `AppSettings.cs`: cross-session shell preferences (the item #5 Unity project path) persisted as a small JSON file under `LocalApplicationData`; `Load` never throws (defaults on a missing/corrupt file), `Save` is best-effort.
- `UI/`: Avalonia views.
  - `MainWindow.cs`: toolbar (New Project / Save / Undo / Redo, tool toggles — Pencil/Eraser/Fill/Line/Rect/Ellipse/Select/Lasso/Stamp/TileMaker/Hitbox — Generate Motion, the color cluster: Eyedropper toggle + RGBA hex color box + preview well + fixed swatch row + persisted recent-colors row, brush/tile size, shape thickness + filled toggle, mirror-symmetry combo, Copy/Paste/Anchor selection buttons, the Resize… dialog button, hitbox key), the item #6 asset-bank panel docked right (sub-element thumbnail list rendered from `SubElement.Pixels` into small BGRA bitmaps — clicking one makes it the stamp tool's active sub-element; a Refresh button; a name box + "Save selection as sub-element" button routed through `EditorShell.SaveSelectionAsSubElement`; and the "Bake pixels" legacy-stamp-mode checkbox), `CanvasView` center, the item #10 timeline strip above the status bar (per-frame toggle buttons over the selected animation, a per-frame duration editor in milliseconds — 0 means "inherit the animation FPS" — routed through `DesignerSession.SetFrameDurationMs`, the onion-skin toggle, — item #31 — the View cluster: Z+/Z−/Reset buttons over the new public zoom API plus the six overlay checkboxes Checker/Pixel grid/Cells/Hitboxes/Attach/Pivots, and — item #33 — the Facing cluster: a "Link 4-Dir" button that converts a directional-capable project into a linked 4-facing project plus four facing toggle buttons that enable only while the project is directional and switch the visible facing through the undoable session command), status bar bottom.
  - `CanvasView.cs`: custom control owning a `ViewportController`. Renders `EditorShell.ActiveCanvasFrame` (the current session frame — or, in map mode, the composed map surface; compositing the session's floating selection through the Core overload outside map mode) via the overlay-aware `RenderToScreenBuffer` (passing `shell.PrepareOverlayContext()`, so the Core overlay passes — checkerboard backdrop, grids, hitboxes, attachment outlines, pivot markers — composite inside Core) into a `WriteableBitmap` (Bgra8888, nearest-neighbor, DPI-aware physical-pixel math) and routes pointer input through `ScreenToCanvas` into `shell.PointerDown/Drag/Up`, redrawing after every pointer event and on every `StateChanged`. With the onion-skin toggle on (non-map mode) it renders through the Core onion overload (`RenderToScreenBuffer(frames, index, OnionSkinSettings, …)`), compositing ghost neighbors under the current frame entirely inside Core. Item #31: `ZoomIn`/`ZoomOut` step through `ViewportController.SetZoom` anchored on the view center and `ResetView` restores the default view.
  - Item #5 map/tileset panel (docked right, `BuildMapPanel`): a Map Mode toggle, the Paint/Erase/Flood action combo, the tile picker (tileset tiles with thumbnails in tile-id order — clicking selects the paint tile), "Add tile from frame" (creates the tileset at the toolbar tile size when absent, then `AddTilesetTileFromCurrentFrame`), map id/size inputs + "Create map", and "Export tileset"/"Export map" buttons. In map mode the canvas shows the Core `MapRenderer` surface and pointer strokes paint/erase map CELLS through `BeginMapPaint`/`PaintMapCell`/`EndMapPaint` (one undoable entry per stroke) or flood-fill on click; Esc cancels the open stroke.
  - `NewProjectDialog.cs`: name + canvas size + category picker over `ToolbarAPI.GetDesignableCategoryDefinitions`, validated with `DesignerPathPolicy.IsSafeFileStem` and the shared canvas limits.
  - `ResizeCanvasDialog.cs`: modal width/height + nine-position anchor picker; OK returns a `ResizeCanvasRequest` the shell routes into `DesignerSession.ResizeCanvas` (one undoable command).
  - Item #3 editor panels (left dock, collapsible `Expander` sections built by `MainWindow.BuildEditorPanels`; each panel owns its own rebuild off `EditorShell` property notifications):
    - `InspectorPanel.cs`: `ToolbarAPI.GetEditableProperties` rendered as kind-appropriate editors (choice combo / ranged spinner / numeric-or-text box / read-only label) with an inline error line fed by the edit's `PropertyEditStatus` and the live per-field validation issue; edits route through `EditorShell.SetProperty`.
    - `LayersPanel.cs`: the current frame's layers (top-most first) with active-layer selection (the layer the tools write to), per-layer visibility/opacity/rename/reorder/duplicate/remove, and an "All frames" toggle that routes add/remove/rename/visibility through the session's cross-frame batch ops.
    - `AnimationsPanel.cs`: animation CRUD + reorder + the selected animation's FPS / loop-range / ping-pong editors (`SetAnimationLoopRange`/`SetAnimationPingPong`), plus frame add/duplicate/remove/reorder (frame selection + per-frame duration stay on the bottom timeline strip).
    - `ProblemsPanel.cs`: the live `ProjectValidationResult` grouped by severity; issues that carry a location render as click-to-focus rows (`EditorShell.FocusProblem` → `SelectFrame`).
    - `PreviewPanel.cs`: `PreviewController` playback on a UI `DispatcherTimer` (honouring per-frame durations + ping-pong), a debounced re-load while stopped, and a disabled Play with the structural-validation reason when the animation cannot play; renders into a reusable `WriteableBitmap`.
    - `LiveDebugPanel.cs`: the persisted Unity-project-path editor, the `StartWatching` toggle (OFF by default, surfaced), a manual `Push to game` (`ExportNowAsync`) with a busy flag + result notice, and the export-scope `LiveDebugSnapshot` status/problems.
    - `PalettePanel.cs`: named-palette CRUD, a swatch grid (click to make a swatch the working color), swatch add/remove/reorder, and the palette-constraint-mode toggle.
    - `ProjectBrowserDialog.cs` / `UnityPathDialog.cs`: the "Open…" browser over `ProjectPersistenceService.ListProjects` (with a restore/discard autosave-recovery prompt) and the Unity-path modal.
    - `BitmapFactory.cs`: shared straight-RGBA → BGRA `WriteableBitmap`/`Image` builder (asset-bank/tileset thumbnails, palette swatches, the preview canvas).
    - `MainWindow` also gained the "Open…" toolbar button and `Ctrl+C`/`Ctrl+V`/`Ctrl+X` selection copy/paste/cut (item #9; a focused `TextBox` consumes its own clipboard keys first).

## Core seams used

- `ToolbarAPI.GetDesignableCategoryDefinitions` / `CreateNewProject` — new-project flow.
- `DesignerSession.Create` — session construction; created on the UI thread so its captured `SynchronizationContext` posts autosave/live-debug events back to the UI thread.
- `session.PointerDown/Drag/Up`, `CancelGesture` (Esc) — one reversible edit per gesture, including selection lift/drag (the session anchors a floating buffer on an outside click and Esc cancels it).
- `session.CopySelection/PasteClipboard/AnchorFloating/ResizeCanvas` — the Copy/Paste/Anchor buttons and the resize dialog.
- `session.Undo/Redo` (Ctrl+Z / Ctrl+Y) and `CommandManager.CommandFailed` — surfaced as a status-bar error flash.
- `session.SaveAsync` (Ctrl+S) — captures an immutable snapshot on the UI thread, serializes on a worker; the shell holds a simple `IsSaving` busy flag that disables Save meanwhile.
- `session.GenerateAnimation(MovingTool)` — the Generate Motion button (regeneration is layer-preserving in Core: only the `Generated` layer is replaced).
- `AssetBankManager` (item #6) — created per project at `<projectDirectory>/AssetBank` and passed into `DesignerSession.Create` as the `ISubElementResolver`, so live debug/export flatten placed attachments; `LoadAllSubElements` feeds the thumbnail list, `LoadFailed` surfaces as the shell's error toast (one bad `.efyvsub` is skipped, the rest load), and `SaveSubElement` backs "Save selection as sub-element" (the floating buffer when one is being dragged, else the masked selection region, else the whole flattened frame). `EditorShell.StampBakePixels` toggles the stamp tool between placing repositionable attachments (default) and the legacy destructive bake.
- `session.SelectFrame` + `session.SetFrameDurationMs` — the timeline strip's frame buttons and duration editor; `ViewportController.ComposeOnionSkin`/onion `RenderToScreenBuffer` + `OnionSkinSettings` (shell defaults) — the onion-skin toggle.
- `EyedropperTool.ColorPicked` + `session.NotifyColorUsed` — completing an eyedropper gesture (composited pick) becomes the working color; every chosen working color lands in the project's persisted recent-colors ring, rendered as the clickable Recent row (`EditorShell.RecentColors`, refreshed from session state publications).
- Item #5 map/tileset seams — `session.CreateTileset/AddTilesetTileFromCurrentFrame` (Add tile from frame), `session.CreateMapSection` (Create map), `session.BeginMapPaint/PaintMapCell/EndMapPaint/FloodFillMapTiles` (map-mode pointer strokes), `MapRenderer` (the map canvas surface), and `ExportEngine.ExportTileset/ExportMap` (the export buttons).
- `ViewportController` — zoom (mouse wheel, anchored; item #31 buttons through `SetZoom`/`ResetView`) and pan (middle-drag); `OnScroll`/`Pan`/`SetZoom`/`SetPan`/`ResetView`/`ScreenToCanvas`/`RenderToScreenBuffer`, plus the item #31 overlay overloads fed by `EditorShell.PrepareOverlayContext()`.
- Item #33 directional seams — `session.EnableDirectionalAuthoring` (the Link 4-Dir button; the shell reports instead of throwing for non-directional projects) and `session.SwitchFacing` (the facing toggles); `EditorShell.IsDirectionalProject`/`ActiveFacing` refresh from every session state publication, so undo/redo of facing commands keeps the buttons coherent. Mirror-generation (`GenerateMirroredFacing`) is session-level only for now.
- `DesignerSessionSnapshot` — status bar (project, tool, frame, zoom, dirty flag, history counts, validation issue counts) and the item #3 panel refresh (problems, animation/layer/palette lists, live-debug snapshot, active-layer re-clamp).
- Item #3 panel seams — the layer/animation/frame CRUD (`Add/Duplicate/Remove/Move/Rename*`, `SetLayerVisibility/Opacity`, the cross-frame `*InAllFrames` batch ops), `ToolbarAPI.GetEditableProperties` + `SetProperty` (inspector), the item #27 lazy `Validation` (problems), `LoadPreview` + `PreviewController` (preview), `LiveDebugController.StartWatching`/`StopWatching`/`ExportNowAsync` + `LiveDebugSnapshot` (live debug), the palette/swatch CRUD + `TrySelectSwatch` + `PaletteConstraintEnabled`/`ActivePaletteIndex` (palette), and the new `ProjectPersistenceService.ListProjects` + `LoadProject`/`LoadAutosave`/`AutosaveExists` (project browser + autosave recovery). The active layer is threaded through the shared tool instances (`ILayerTool.ActiveLayerIndex`) and into `CopySelection`/`PasteClipboard`.

## Threading rules

`DesignerSession` is synchronous and not thread-safe. Every session call in this app happens on the Avalonia UI thread. The only asynchronous seams are persistence saves (`SaveAsync`, plus the core's own debounced autosave/live-debug workers, whose events re-enter through the session's captured synchronization context).

## Known MVP limits (deliberate, later batches)

- Placed attachments render as OUTLINES + pivot anchors on the canvas (the item #31 Attach/Pivots overlays); their actual pixels still only appear in the exported/live-debug atlas — a full attachment pixel preview is still deferred. They remain fully repositionable by anchor-grabbing with the stamp tool, and the attachment CRUD surface (`AddAttachment`/`MoveAttachment`/`RemoveAttachment`) is still session-level only (no attachment list panel).
- `Ctrl+X` cut = copy-to-clipboard + LIFT the selection off the active layer into a floating buffer (the pixels detach; drag+click relocates them, Esc restores). A committed erase-in-place (delete the selected region, leaving a hole, as one undoable command) is a Core-side gap: no region-clear command exists on `DesignerSession` and Core edits were restricted to `ListProjects` for this item, so it is deferred.
- Setting the Unity project path is not undoable and does not dirty-mark the project (it is a config field, not document content); it is captured by the next save, read live by export/live-debug validation, and remembered cross-session as an app setting. Directory picking is a plain text box (no native folder dialog).
- The item #5 map/tileset panel is deliberately minimal (pick/paint/erase/flood + create/export): no tile RE-capture or rename/remove buttons (the session surface exists) and no prop-placement UI (`ApplyMapTool` scatter/noise is session-level only). Item #31 closed the zoom-control and grid-overlay gaps: Z+/Z−/Reset drive the public `SetZoom`/`ResetView` API and the Cells overlay draws the map/tileset cell boundaries.
- The item #7 destructive layer filters (`ToolbarAPI.GetLayerFilters` + the session `Apply*Filter` surface) and the item #7 effect-descriptor CRUD are not exposed as panels yet (session-level only).
- Effect/loop editing: the animations panel exposes FPS, loop range, and ping-pong; per-animation effect descriptors and onion-skin depth controls are not surfaced.
